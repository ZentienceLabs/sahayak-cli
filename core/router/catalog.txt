# Sahayak intent catalog — DATA, not code.
#
# This is the data-driven half of the semantic router. To teach Sahayak a new way
# of phrasing a request, ADD A LINE here — no Go, no recompiling a regex. Each
# request is matched by MEANING to the nearest example below (semantic similarity),
# and routed to that example's playbook kind; Go then fills the slots deterministically
# (playbook.BuildPlan) and only fires if every required slot is grounded in the request.
#
# Format:
#   intent <name> <kind>      starts an intent block; <kind> must be a known playbook
#                             kind: list | logs | image | rollout | restart |
#                             verifyenv | searchcfg
#   - <example phrase>        an example phrasing for that intent (use X/Y as a
#                             stand-in for the app/keyword the user will name)
#
# Blank lines and lines beginning with # are ignored. The default catalog ships
# embedded; a user file (SAHAYAK_CATALOG=/path) is appended on top of it.
